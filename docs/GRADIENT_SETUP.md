# Setting up GRADIENT_API_KEY and GRADIENT_API_URL

ShadowLab uses **DigitalOcean Gradient™ AI** for adversarial prompt generation and vulnerability analysis when these are set. Follow these steps to enable it.

---

## 1. Get your API token (GRADIENT_API_KEY)

1. In the DigitalOcean dashboard, go to **Applications & API** → **Tokens** (the page you’re on).
2. Click **Generate New Token**.
3. Give the token a name (e.g. `ShadowLab Gradient`).
4. Choose the scopes you need for Gradient AI (see [DigitalOcean Gradient AI docs](https://docs.digitalocean.com/products/ai/) for the required scope, if specified).
5. Click **Generate Token**.
6. **Copy the token immediately** – it’s shown only once. This value is your **GRADIENT_API_KEY**.

> **Note:** If Gradient™ AI uses a separate key (e.g. from a Gradient-specific section in the console), use that key instead. Check the [Gradient AI documentation](https://mlh.link/digitalocean-gradient) for the current auth method.

---

## 2. Set GRADIENT_API_URL (optional)

ShadowLab defaults to:

```text
GRADIENT_API_URL=https://api.gradient.digitalocean.com/v1
```

Only set this if DigitalOcean documents a different base URL for your region or product. The backend appends paths like `/completions` to this base.

---

## 3. Configure the backend

In the **backend** directory:

1. Copy the example env file:
   ```bash
   cd backend
   cp .env.example .env
   ```

2. Edit `.env` and set your token:
   ```env
   GRADIENT_API_KEY=your_token_here
   GRADIENT_API_URL=https://api.gradient.digitalocean.com/v1
   ```

3. Restart the backend so it picks up the new env:
   ```bash
   uvicorn app.main:app --reload
   ```

Do **not** commit `.env` (it’s in `.gitignore`). Keep your token secret.

---

## 4. Verify

- Run a scan from the dashboard. If Gradient is used, the Security Report will show:  
  **“Adversarial attacks generated by DigitalOcean Gradient™ AI.”**
- Without a valid key or with the wrong URL, ShadowLab falls back to seed attacks and the heuristic judge (no error; it just won’t use Gradient).

---

## Troubleshooting

| Issue | What to do |
|-------|------------|
| Gradient never used / label never appears | Confirm `GRADIENT_API_KEY` is set in `backend/.env` and the backend was restarted. |
| 401 Unauthorized | Token may be wrong or expired. Generate a new token and update `.env`. |
| 404 or connection errors | Check [Gradient AI docs](https://mlh.link/digitalocean-gradient) for the correct **GRADIENT_API_URL** and endpoint path. |
| Timeouts | Gradient may be slow; the client uses 30s for generation and 15s for analysis. |

For the exact API base URL, auth header, and request format, always refer to the official **DigitalOcean Gradient™ AI** documentation.
