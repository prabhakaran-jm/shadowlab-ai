# Setting up GRADIENT_API_KEY and GRADIENT_API_URL

ShadowLab uses **DigitalOcean Gradient™ AI Serverless Inference** for adversarial prompt generation and vulnerability analysis. You need a **Model Access Key** (not the general DigitalOcean API token).

---

## 1. Get your Model Access Key (required for Gradient)

The key must be from **Gradient AI → Serverless Inference**, not from **Applications & API → Tokens**.

1. In the DigitalOcean Control Panel, go to **Gradient AI Platform** → **Serverless Inference**  
   (or open: [Serverless Inference](https://cloud.digitalocean.com/gradient-ai-platform/serverless-inference)).
2. Scroll to **Model Access Keys**.
3. Click **Create Access Key** (or copy an existing key).
4. **Copy the key** and set it in `backend/.env`. ShadowLab accepts any of:
   - `MODEL_ACCESS_KEY` (same as in the [official Gradient SDK docs](https://docs.digitalocean.com/products/gradient-ai-platform/how-to/use-serverless-inference))
   - `GRADIENT_MODEL_ACCESS_KEY`
   - `GRADIENT_API_KEY`

If you only have a token from **Applications & API → Tokens**, that is the general DO API token. It does **not** authenticate Serverless Inference. You must create a Model Access Key in the Gradient AI section.

---

## 2. Set GRADIENT_API_URL (optional)

ShadowLab defaults to the Serverless Inference API:

```text
GRADIENT_API_URL=https://inference.do-ai.run
```

The client calls `POST {GRADIENT_API_URL}/v1/chat/completions`. Only change this if DigitalOcean documents a different base URL.

---

## 3. Configure the backend

In the **backend** directory:

1. Copy the example env file:
   ```bash
   cd backend
   cp .env.example .env
   ```

2. Edit `.env` and set your **Model Access Key**:
   ```env
   GRADIENT_MODEL_ACCESS_KEY=your_model_access_key_here
   GRADIENT_API_URL=https://inference.do-ai.run
   ```
   You can use `GRADIENT_API_KEY` instead of `GRADIENT_MODEL_ACCESS_KEY` (same value).

3. Restart the backend:
   ```bash
   uvicorn app.main:app --reload
   ```

Do **not** commit `.env` (it’s in `.gitignore`).

---

## 4. Verify

- Run a scan from the dashboard. In the logs you should see `POST https://inference.do-ai.run/v1/chat/completions` (or your custom base URL).
- The Security Report should show: **“Adversarial attacks generated by DigitalOcean Gradient™ AI.”**
- Without a valid Model Access Key, ShadowLab falls back to seed attacks and the heuristic judge (no crash).

### Test the analysis model (Llama 3.3 70B)

To see the **vulnerability analysis** model in action (reason + suggested fix from Gradient):

1. In the dashboard, set **API Endpoint** to: `http://localhost:8000/mock-vulnerable-api`
2. Set **Target Description** to anything (e.g. `Mock vulnerable API`).
3. Click **Start Scan**.

The mock endpoint returns a response containing “system prompt”, so the judge marks it as **fail**. That triggers a second Gradient call (analysis). In the logs you should see **two** requests to `inference.do-ai.run`: one for prompt generation, one for vulnerability analysis. The report will show a **fail** row and a **Recommended Fix** from Gradient.

---

## 5. Optional: change models (two-model design)

ShadowLab uses two models by default:

| Use case | Default model | Env var |
|----------|----------------|---------|
| Adversarial prompt generation | `openai-gpt-oss-20b` (lightweight) | `GRADIENT_MODEL_GENERATION` |
| Vulnerability / failure analysis | `llama3.3-70b-instruct` (stronger) | `GRADIENT_MODEL_ANALYSIS` |

Override in `.env` if needed:

```env
GRADIENT_MODEL_GENERATION=openai-gpt-oss-20b
GRADIENT_MODEL_ANALYSIS=llama3.3-70b-instruct
```

Available models: [DigitalOcean Gradient AI – Available models](https://docs.digitalocean.com/products/gradient-ai-platform/details/models). You can also call `GET https://inference.do-ai.run/v1/models` with your Model Access Key to list models.

---

## Troubleshooting

| Issue | What to do |
|-------|------------|
| No Gradient requests in logs / label never appears | Use a **Model Access Key** from Gradient AI → Serverless Inference, not the DO API token. Ensure it’s set as `GRADIENT_MODEL_ACCESS_KEY` or `GRADIENT_API_KEY` in `backend/.env` and restart the backend. |
| 401 Unauthorized | Key may be wrong or revoked. Create a new Model Access Key in Serverless Inference and update `.env`. |
| 404 or connection errors | Confirm `GRADIENT_API_URL` is `https://inference.do-ai.run` (no trailing slash). |
| Timeouts | Increase timeout in code or try a smaller model (e.g. `GRADIENT_MODEL=llama3-8b-instruct`). |

For the latest API details, see [DigitalOcean Gradient AI – Serverless Inference](https://docs.digitalocean.com/products/gradient-ai-platform/how-to/use-serverless-inference).
