# ShadowLab Demo Script (Optimized for Judges)

Use this flow for your ~3-minute hackathon demo video and live judging.

---

## Step 1 – Introduce the problem

*“AI APIs often fail due to prompt injection and hidden vulnerabilities. Developers rarely test for these until it’s too late.”*

---

## Step 2 – Open ShadowLab dashboard

- Show the title: **ShadowLab – AI Adversarial API Tester**
- Point out the tagline and the **“Powered by DigitalOcean Gradient™ AI”** line so judges see sponsor integration.

---

## Step 3 – Start a scan

**Option A – Real API (recommended for impact):**  
- API Endpoint: `https://inference.do-ai.run/v1/chat/completions`  
- Request body format: **messages (OpenAI / Gradient)**  
- Model: `llama3.3-70b-instruct`  
- Target API key: your Gradient Model Access Key (optional field)  
- Target description: e.g. *“Gradient Serverless Inference API”*  
- See [docs/DEMO_REAL_API.md](DEMO_REAL_API.md) for full steps.

**Option B – Built-in mock (no key):**  
- API Endpoint: `http://localhost:8000/mock-vulnerable-api` (with `ALLOW_LOCALHOST_TARGET=1` in backend `.env`)  
- Target description: e.g. *“Local mock API”*

- Click **Start Scan**.
- As the live attack console runs, say: *“ShadowLab uses **DigitalOcean Gradient™ AI** to generate adversarial prompts when configured, and runs them against the target API.”*

---

## Step 4 – Security report

- When the report appears, highlight:
  - **Safety score** (e.g. 70/100)
  - **Vulnerability summary** (e.g. “Prompt Injection: 1 detected”)
- Say: *“The judge detects failures and, when Gradient is configured, uses **Gradient AI** to analyze responses and explain why they’re unsafe.”*

---

## Step 5 – Fix recommendations (show the “wow” factor)

- Scroll to **Recommended Fixes** and keep this section visible—it’s the main differentiator.
- Say: *“**Gradient AI** analyzes the response and suggests fixes so developers know exactly how to harden their API.”*
- If the run used Gradient-generated attacks, point out the line: *“Adversarial attacks generated by DigitalOcean Gradient™ AI.”*

---

## Step 6 – Conclusion

*“ShadowLab helps developers ship safer AI systems before attackers find these vulnerabilities. Attack generation and vulnerability analysis are both powered by **DigitalOcean Gradient™ AI**.”*

---

- **Frontend:** DigitalOcean App Platform  
- **Backend:** DigitalOcean App Platform  
- **AI engine:** DigitalOcean Gradient™ AI  
- **Repository:** GitHub (public)  
- **Demo video:** [Add your ~3 min YouTube/Vimeo/Facebook URL before submitting]  
- **Live demo (optional):** [Add your deployed app URL]

This aligns the narrative with the hackathon’s **Technological Implementation** criteria and clearly shows Gradient AI usage.
