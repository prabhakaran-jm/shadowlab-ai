# Testing with a Real API During the Demo

Use one of these options to run ShadowLab against a real API in your video or live demo.

---

## Option 1: DigitalOcean Gradient™ AI as the scan target (recommended for hackathon)

Use **Gradient Serverless Inference** as the API under test. You need a **Model Access Key** (same as for ShadowLab’s own Gradient usage).

1. **In the dashboard**
   - **API Endpoint:** `https://inference.do-ai.run/v1/chat/completions`
   - **Target Description:** e.g. *“Gradient chat completions API”*
   - **Request body format:** **messages (OpenAI / Gradient)**
   - **Model:** e.g. `llama3.3-70b-instruct` or `openai-gpt-oss-20b` (see [Gradient models](https://docs.digitalocean.com/products/gradient-ai-platform/details/models))
   - **Target API key (optional):** paste your **Model Access Key** (the key you use for ShadowLab’s backend env). It is sent only with the scan request and is not stored.

2. Click **Start Scan**. ShadowLab will:
   - Use Gradient to **generate** adversarial prompts (if `GRADIENT_MODEL_ACCESS_KEY` is set on the backend),
   - Send those prompts to **Gradient again** as the target API (using the key you entered),
   - Judge the responses and optionally use Gradient to **analyze** failures and suggest fixes.

This shows a full loop: Gradient generates attacks → Gradient is the target → Gradient helps explain findings.

---

## Option 2: Built-in mock (no external API)

- **API Endpoint:** `http://localhost:8000/mock-vulnerable-api` (with backend running).
- Set **ALLOW_LOCALHOST_TARGET=1** in `backend/.env` so the SSRF guard allows localhost.
- No API key needed. The mock returns a response that contains “system prompt”, so you’ll get at least one finding and can show the report and Gradient-generated fix (if the backend has a Gradient key).

---

## Option 3: Any OpenAI-compatible API with a key

If you have another OpenAI-style API (e.g. OpenAI, Groq, Together):

- **API Endpoint:** the chat completions URL (e.g. `https://api.openai.com/v1/chat/completions`).
- **Request body format:** **messages (OpenAI / Gradient)**.
- **Model:** the model id (e.g. `gpt-4o-mini`).
- **Target API key:** `Bearer <your-api-key>` or just the key (the UI adds “Bearer ”).

The target API must accept `POST` with JSON `{ "model": "...", "messages": [{"role":"user","content":"..."}] }` and return a response we can parse (we extract assistant content from JSON when possible).

## Quick demo script (real API)

1. Set **backend** `GRADIENT_MODEL_ACCESS_KEY` so ShadowLab generates attacks with Gradient.
2. In the UI, set **API Endpoint** to `https://inference.do-ai.run/v1/chat/completions`, **body format** to **messages**, **Model** to `llama3.3-70b-instruct`, and **Target API key** to your Model Access Key.
3. **Target description:** e.g. *“Gradient Serverless Inference chat API”*.
4. Click **Start Scan** and narrate: *“We’re scanning the same Gradient API we use for attack generation—so we’re testing our own AI stack.”*
5. When the report appears, highlight the safety score, any findings, and the “Adversarial attacks generated by DigitalOcean Gradient™ AI” badge and Gradient-generated fix suggestions.

## Showing findings and a lower safety score (for the video)

When you scan a well-hardened API (e.g. Gradient), you often get 100/100 and “0 detected”. To **show real findings and a lower score** in your demo:

1. **Use the built-in mock-vulnerable endpoint** as the scan target. It always returns a response containing “system prompt”, so the judge will mark it as a failure and the safety score will drop (e.g. 80 or lower depending on how many attacks hit it).

2. **In the dashboard (deployed app):**
   - **API Endpoint:** `https://shadowlab-h9yu6.ondigitalocean.app/api/mock-vulnerable-api`  
     (replace with your own app URL if different; the backend is under `/api` when using the default ingress.)
   - **Target Description:** e.g. *“Mock vulnerable API for demo”*
   - **Request body format:** message (default)
   - Leave **Model** and **Target API key** empty.

3. Click **Start Scan**. You should see:
   - At least one **fail** in Findings (e.g. “Response contains sensitive phrase: 'system prompt'”).
   - **Vulnerability summary** with 1+ detected (e.g. System Prompt Extraction).
   - **Lower Global Safety Score** (e.g. 80 or 60).
   - **Recommended Fixes** with Gradient-generated suggestions (if `GRADIENT_MODEL_ACCESS_KEY` is set).

Use this for the “here’s what a vulnerable API looks like” part of your video; then switch to the real Gradient URL to show a healthy 100/100.

---

## Security note

The **Target API key** is sent in the request body to your backend and then used only for the outbound requests to the target URL. It is not stored. For production you’d use a dedicated “target credentials” store or per-target secrets; for the demo, pasting the key in the optional field is acceptable.
